NI-R Data Model Overview

The NIDASH Data Model for neuroimaging results (NI-R Data Model) is concerned with the modelling of model fitting and inference in the context of massively univariate analyses. A typical example is the analysis of functional MRI data but studies involving other modalities (such as PET) and sequences (e.g. anatomical MRI through VBM) can also be modelled. The domain covered by NI-R data model is represented in Fig. 1.

NI-R Core Structures

The concepts found in the core of NI-Results are introduced in the rest of this section. They are summarized in Table 2, where they are categorized as type or relation. The first column lists concepts, the second column indicates whether a concept maps to a type or a relation, whereas the third column contains the corresponding name, as it appears in Figure 1. Names of relations have a verbal form in the past tense to express what happened in the past, as opposed to what may or will happen. In the core of PROV, all relations are binary.

NIDASH Results Data Model Types and Relations.

Element 1: Model Fitting

ModelParametersEstimation

A ModelParametersEstimation ◊is a the process of estimating the parameters of a general linear model from the available data (sub-type of prov:Activity).

A ModelParametersEstimation ◊ has:
  • prov:label: an OPTIONAL label describing the ModelParametersEstimation.

Data

A Data ◊is a "A collection or single item of factual information, derived from measurement or research, from which conclusions may be drawn." (sub-type of prov:Entity).

A Data ◊ has:
  • prov:label: an OPTIONAL label describing the Data.
  • grandMeanScaling: an OPTIONAL binary flag defining whether the grand mean of the data was scaled.
  • targetIntensity: an OPTIONAL value to which the grand mean of the Data was scaled (applies only if grandMeanScaling is true).
  • medianIntensity: an OPTIONAL value corresponding to the median intensity of the Data.

DesignMatrix

A DesignMatrix ◊is a design matrix associated with a particular design (sub-type of prov:Entity).

A DesignMatrix ◊ has:
  • prov:label: an OPTIONAL label describing the DesignMatrix.
  • prov:location: an OPTIONAL path to a csv file describing the design matrix.
  • visualisation: an OPTIONAL graphical representation of an entity.
  • originalFileName: an OPTIONAL original name associated with a file (without path) in the context of the analysis software package.

ParameterEstimateMap

A ParameterEstimateMap ◊is a a map whose value at each location is the estimate of a model parameter (sub-type of prov:Entity).

A ParameterEstimateMap ◊ has:
  • prov:label: an OPTIONAL label describing the ParameterEstimateMap.
  • prov:location: an OPTIONAL path to the corresponding image file.
  • originalFileName: an OPTIONAL original name associated with a file (without path) in the context of the analysis software package.
  • atCoordinateSpace: an OPTIONAL attribute between a SpatialImage entity and a CoordinateSpace entity that is used to associate spatial metadata with a physical file.
  • crypto:sha512: an OPTIONAL SHA512 hash for the image file.

Element 2: Inference

Inference

A Inference ◊is an Activity defined as The process of inferring the excursion set from a statistical map..

A Inference ◊ has:
  • prov:label: an OPTIONAL label describing the Inference.

ExcursionSet

A ExcursionSet ◊is an Entity defined as Set of map elements surviving a thresholding procedure..

A ExcursionSet ◊ has:
  • prov:label: an OPTIONAL label describing the ExcursionSet.
  • prov:location: an OPTIONAL Path to the corresponding image file..
  • clusterLabelsMap: an OPTIONAL A map whose value at each location denotes cluster membership. Each cluster is denoted by a different integer..
  • originalFileName: an OPTIONAL Original name associated with a file (without path) in the context of the analysis software package..
  • atCoordinateSpace: an OPTIONAL Attribute between a SpatialImage entity and a CoordinateSpace entity that is used to associate spatial metadata with a physical file..
  • crypto:sha512: an OPTIONAL SHA512 hash for the image file..

ClusterStatistic

A ClusterStatistic ◊is an Entity defined as Statistic defined at the cluster-level in an excusion set. In SPM it is defined as the cluster size. .

A ClusterStatistic ◊ has:
  • prov:label: an OPTIONAL label describing the ClusterStatistic.
  • clusterSizeInVoxels: an OPTIONAL Number of voxels that make up the cluster..
  • clusterLabelId: an OPTIONAL Integer associated with a particular cluster as specified in the clusterLabelsMap..
  • clusterSizeInResels: an OPTIONAL Number of resels that make up a cluster..
  • pValueUncorrected: an OPTIONAL A p-value reported without correction for multiple testing. .
  • pValueFWER: an OPTIONAL "A quantitative confidence value resulting from a multiple testing error correction method which adjusts the p-value used as input to control for Type I error in the context of multiple pairwise tests".
  • qValueFDR: an OPTIONAL P-value adjusted for the multiple testing, controlling for the False Discovery Rate.

PeakStatistic

A PeakStatistic ◊is an Entity defined as Statistic defined at the peak-level in an excursion set..

A PeakStatistic ◊ has:
  • prov:label: an OPTIONAL label describing the PeakStatistic.
  • prov:location: an OPTIONAL Path to the corresponding image file..
  • equivalentZStatistic: an OPTIONAL Statistic value transformed into Z units; the output of a process which takes a non-normal statistic and transforms it to an equivalent z score..
  • pValueUncorrected: an OPTIONAL A p-value reported without correction for multiple testing. .
  • pValueFWER: an OPTIONAL "A quantitative confidence value resulting from a multiple testing error correction method which adjusts the p-value used as input to control for Type I error in the context of multiple pairwise tests".
  • qValueFDR: an OPTIONAL P-value adjusted for the multiple testing, controlling for the False Discovery Rate.
